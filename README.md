# VR Racing UnityApp

## Описание проекта

Проект представляет собой гоночную игру, созданную в Unity. Игрок управляет машиной на 3D-трассе и соревнуется с несколькими интеллектуальными соперниками. Перед гонкой трасса может перестраиваться процедурно: создается круглая, овальная или плавная змейкообразная форма с разным размером. Соперники двигаются по актуальной трассе с использованием алгоритма A*, который строит маршрут по системе waypoint-точек.

В игре есть стартовое меню, настройки гонки, звук двигателя, подсчет кругов, рейтинг позиции игрока, стартовая/финишная линия, парковка машин после финиша, запеченный свет и Reflection Probe.

## Как работает игра

После запуска сцены появляется меню. В нем можно выбрать:

- количество кругов;
- количество соперников;
- режим управления;
- громкость двигателя.

После нажатия кнопки `Start Race` генерируется трасса и начинается гонка. Игрок управляет красной машиной, а синие машины являются соперниками. Во время гонки на экране отображаются:

- текущий круг;
- позиция игрока в рейтинге;
- время заезда;
- статус финиша.

 Когда машина проходит заданное количество кругов, для нее фиксируется финиш. Соперники после завершения гонки уезжают на парковочные точки за трассой. Игрок может открыть меню заново через кнопку `Menu / Restart`.

Если машина застряла или выехала неудобно, можно нажать `R`, чтобы вернуть ее на ближайшую точку трассы.

## Управление

- `W` или стрелка вверх - газ.
- `S` или стрелка вниз - тормоз / движение назад.
- `A` или стрелка влево - поворот влево.
- `D` или стрелка вправо - поворот вправо.
- `R` - вернуть машину на трассу.

## Режимы управления

В меню можно выбрать один из трех режимов:

- `Arcade` - самый быстрый и резкий режим.
- `Comfort` - более спокойное управление, меньше резких поворотов.
- `VR` - наиболее плавный режим, подходящий для VR-управления.

Режим влияет на ускорение, максимальную скорость, поворот и сцепление машины.

## Процедурная трасса

Перед стартом гонки создается новая трасса.

Дорога создается как единое mesh-полотно. Бордюры по краям также строятся как непрерывные ленты. По внешним границам добавляются коллизии, чтобы машины не выезжали за пределы трассы. Под трассой создается увеличенный зеленый пол, чтобы все варианты трассы помещались на сцене.

После генерации создается новый набор waypoint-точек. Эти точки используются для подсчета прогресса, кругов и построения маршрутов соперников.

## Интеллектуальные соперники и A*

Для движения соперников используется waypoint-граф трассы. Он состоит из точек, расположенных по маршруту гонки. Так как трасса генерируется заново, граф тоже перестраивается под новую форму.

Алгоритм A* используется для поиска пути между точками графа. Соперник берет ближайший узел как старт, выбирает следующую цель на трассе и получает маршрут через A*. После прохождения части маршрута путь пересчитывается снова, поэтому движение остается привязанным к текущему графу трассы.

В реализации A* используются:

- открытый список узлов;
- стоимость пути до узла (`gScore`);
- примерная стоимость пути до цели (`fScore`);
- словарь предыдущих узлов (`cameFrom`);
- эвристика на основе расстояния между точками.

После нахождения маршрута соперник следует по точкам трассы с учетом небольшого смещения по полосе. Это позволяет машинам двигаться не хаотично, а по рассчитанному маршруту.

## Основные файлы проекта

### Сцены

```text

Assets/Scenes/SampleScene.unity
```

Основные сцены проекта. В них находятся машины, меню, свет, Reflection Probe и игровые объекты. Текущая трасса создается во время запуска гонки.

### Управление игроком

```text
Assets/Scripts/PlayerCarController.cs
```

Отвечает за управление машиной игрока: ускорение, торможение, повороты, сцепление и разные режимы управления.

### Соперники

```text
Assets/Scripts/RacingAIDriver.cs
```

Отвечает за движение AI-соперников по трассе.

### Алгоритм A*

```text
Assets/Scripts/AStarGrid.cs
```

Содержит граф трассы и реализацию поиска пути A*.

### Игровая логика

```text
Assets/Scripts/RaceGameManager.cs
```

Отвечает за старт гонки, рестарт, запуск генерации трассы, отображение HUD, время гонки и позицию игрока.

```text
Assets/Scripts/RaceProgressTracker.cs
```

Отслеживает прогресс каждой машины, количество кругов и финиш. Круг фиксируется после прохождения половины трассы и пересечения стартовой/финишной линии.

```text
Assets/Scripts/ProceduralTrackGenerator.cs
```

Создает процедурную трассу, бордюры, коллизии, большой пол, waypoint-граф и парковочные точки.

```text
Assets/Scripts/TrackResetZone.cs
```

Позволяет вернуть машину на трассу по клавише `R`.

### Меню

```text
Assets/Scripts/MainMenuController.cs
```

Отвечает за меню настроек: количество кругов, количество соперников, режим управления и громкость.

```text
Assets/Scripts/ProceduralEngineAudio.cs
```

```text
Assets/Models/BlenderCreatedRacer.obj
```

## Свет и Reflection Probe

В сцене используется запеченный свет. Для освещения добавлен направленный источник света `Baked Sun`. Также используется `Track Reflection Probe`, который отвечает за отражения и улучшает внешний вид объектов на трассе.

Для запекания света можно использовать пункт меню Unity:

```text
Window > Rendering > Lighting > Generate Lighting
```

После запекания Unity сохраняет данные освещения в сцене.
