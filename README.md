# VR Racing UnityApp

## Описание проекта

Проект представляет собой гоночную игру, созданную в Unity. Игрок управляет машиной на 3D-трассе и соревнуется с несколькими интеллектуальными соперниками. Соперники двигаются по трассе с использованием алгоритма A*, который строит маршрут по системе waypoint-точек.

В игре есть стартовое меню, настройки гонки, звук двигателя, подсчет кругов, рейтинг позиции игрока, стартовая линия, финишная логика и запеченный свет с Reflection Probe.

## Как работает игра

После запуска сцены появляется меню. В нем можно выбрать:

- количество кругов;
- количество соперников;
- режим управления;
- громкость двигателя.

После нажатия кнопки `Start Race` начинается гонка. Игрок управляет красной машиной, а синие машины являются соперниками. Во время гонки на экране отображаются:

- текущий круг;
- позиция игрока в рейтинге;
- время заезда;
- статус финиша.

Когда машина проходит заданное количество кругов, для нее фиксируется финиш. Соперники после завершения гонки уезжают на парковочные точки за трассой. Игрок может открыть меню заново через кнопку `Menu / Restart`.

Если машина застряла или выехала неудобно, можно нажать `R`, чтобы вернуть ее на ближайшую точку трассы.

## Управление

- `W` или стрелка вверх - газ.
- `S` или стрелка вниз - тормоз / движение назад.
- `A` или стрелка влево - поворот влево.
- `D` или стрелка вправо - поворот вправо.
- `R` - вернуть машину на трассу.

## Режимы управления

В меню можно выбрать один из трех режимов:

- `Arcade` - самый быстрый и резкий режим.
- `Comfort` - более спокойное управление, меньше резких поворотов.
- `VR` - наиболее плавный режим, подходящий для VR-управления.

Режим влияет на ускорение, максимальную скорость, поворот и сцепление машины.

## Интеллектуальные соперники и A*

Для движения соперников используется waypoint-граф трассы. Он состоит из точек, расположенных по маршруту гонки. Соперники ориентируются на эти точки и двигаются от одной точки к следующей.

Трасса может перестраиваться перед стартом гонки. При генерации создается новое общее полотно дороги, непрерывные бордюры по двум сторонам и новый набор waypoint-точек. После этого граф передается в систему A*, поэтому соперники каждый раз строят маршрут уже по актуальной форме трассы.

Алгоритм A* используется для поиска пути между точками графа. В реализации используются:

- открытый список узлов;
- стоимость пути до узла (`gScore`);
- примерная стоимость пути до цели (`fScore`);
- словарь предыдущих узлов (`cameFrom`);
- эвристика на основе расстояния между точками.

После нахождения маршрута соперник следует по точкам трассы. Это позволяет машинам двигаться не хаотично, а по заранее рассчитанному маршруту.

## Свет и Reflection Probe

В сцене используется запеченный свет. Для освещения добавлен направленный источник света `Baked Sun`. Также используется `Track Reflection Probe`, который отвечает за отражения и улучшает внешний вид объектов на трассе.

Для запекания света можно использовать пункт меню Unity:

```text
Window > Rendering > Lighting > Generate Lighting
```

После запекания Unity сохраняет данные освещения в сцене.
